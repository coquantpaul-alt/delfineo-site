This page is both documentation and a rendered preview. The file lives at `src/content/research/en/visuals-demo.mdx` — open it alongside the rendered page to see how each component is authored.

The components are available automatically in any `.mdx` article; you do **not** need to import anything. If you're writing a plain `.md` article (as most news briefs are today), you can rename the file to `.mdx` to use any of the components below.

## Figure — images with caption and source

Use `<Figure>` for photos and static diagrams. The `width` prop controls how far the image breaks out of the text column: `inset` is narrower than the body, `normal` matches the body (default), `wide` extends ~60 px either side, and `full` breaks out to the viewport.

Images live in `public/images/<collection>/<slug>/<filename>`. Reference them as `/images/...` (no leading `public`). Both PNG and SVG are supported.

## Chart — static chart images

`<Chart>` is a thin wrapper around `<Figure>` that adds a small "CHART" eyebrow and a tinted background, so exported chart images read as data-viz artefacts rather than photos. Use it for charts exported from Excel, Datawrapper, Flourish, or any external tool.

  alt="Sample exported chart"

  width="wide"
/>

## InteractiveChart — Chart.js line / bar / scatter

`<InteractiveChart>` renders a live Chart.js instance. Chart.js is loaded from the CDN only when at least one interactive chart is on the page, so articles without one pay nothing. Supported types today: `line`, `bar`, `scatter`, `doughnut`.

<InteractiveChart
  type="line"
  caption="Cellnex revenue, 2019–2024 (€bn)"
  source="Company filings"
  data={{
    labels: ['2019', '2020', '2021', '2022', '2023', '2024'],
    datasets: [
      { label: 'Revenue', data: [1.04, 1.61, 2.54, 3.50, 3.93, 4.11] },
      { label: 'EBITDA adj.', data: [0.77, 1.22, 1.95, 2.62, 2.94, 3.10] },
    ],
  }}
/>

Pass a second dataset and the legend appears automatically. Colours follow Delfineo's palette (teal → gold → forest → wine → rust → slate) in order. Override any colour by setting `borderColor` or `backgroundColor` on the dataset.

<InteractiveChart
  type="bar"
  caption="Share of Q1 2026 investment-grade issuance, by bank"
  source="Bloomberg"
  data={{
    labels: ['JPM', 'BofA', 'Citi', 'GS', 'MS', 'Wells'],
    datasets: [{ label: 'USDbn', data: [82, 61, 54, 42, 38, 22] }],
  }}
/>

## DataTable — financial-register tables

Wrap a Markdown table in `<DataTable>` for tight rows, numeric alignment, and thin rules. Right-align numeric columns in the Markdown header (using `---:`) so the alignment survives the render.

<DataTable caption="Cellnex — simplified FY results (€m)" source="Company filings · Bloomberg">

| €m            |  2022 |  2023 |  2024 |
| ------------- | ----: | ----: | ----: |
| Revenue       | 3,503 | 3,925 | 4,110 |
| EBITDA adj.   | 2,621 | 2,935 | 3,097 |
| Capex         | 2,180 | 1,150 |   900 |
| Net debt      |18,180 |17,450 |16,300 |

</DataTable>

Pass `compact` for tighter rows when you have many lines.

## Callout — emphasized takeaways

Three variants: `insight` (navy, default), `risk` (wine), `note` (muted). Use sparingly — one per article max.

<Callout type="insight" title="Key insight">
  Cellnex's next three years of committed capex are already covered by contracted cash flow — the business has stopped being a capital-raising story and become a de-leveraging one.
</Callout>

<Callout type="risk" title="Risk">
  Roughly 22% of Cellnex revenue is tied to a single counterparty (Orange). Contract renewals in 2027–2028 are the main downside path.
</Callout>

<Callout type="note" title="Note">
  All figures are management-reported and not IFRS-adjusted for lease accounting changes in 2024.
</Callout>

## PullQuote — emphasized sentence

For short, quotable lines — one or two sentences, rarely more. Attribution is optional.

<PullQuote attribution="— Tobias Martinez, Cellnex CEO, Q4 2024 call">
  The moat isn't the towers. It's the twenty-year leases.
</PullQuote>

## Plain Markdown still works

All the existing Markdown syntax — `**bold**`, `*italic*`, `[links](/somewhere)`, bullets, `inline code`, `## subheadings`, `> blockquotes` — keeps working exactly as before. The components above sit alongside Markdown, not in place of it.

> Plain blockquotes still render. Use them for third-party quotes in the flow of prose; reserve `<PullQuote>` for visually emphasized moments.
